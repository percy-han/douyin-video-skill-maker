# 🧠 认知框架合成模块

从所有视频转录中提取认知框架：心智模型、决策启发式、表达DNA、价值观。

> 借鉴Nuwa方法论，提取的不是"他说过什么"，而是"他怎么想"。

---

## 💡 核心理念

### 传统方案的问题 ❌

```
用户："当前美联储可能降息，对A股有什么影响？"

传统RAG：
→ 检索"美联储降息"相关视频
→ 返回："刘德超在2026-04-03说：'如果降息，A股可能...'"
→ 问题：这是3个月前的观点，经济环境已变化
→ 价值：低（过时信息）
```

### 认知复刻方案 ✅

```
用户："当前美联储可能降息，对A股有什么影响？"

认知复刻：
→ 提取刘德超分析"降息→股市"的思维框架
→ 识别他关注的关键变量（通胀、汇率、估值）
→ 应用他的分析逻辑到当前数据
→ 输出："基于刘德超的分析方法，当前情况下..."

价值：高（可应用于新情况）
```

---

## 🏗️ Nuwa的核心方法（我们借鉴的）

### 1. 三重验证机制 ⭐⭐⭐

一个观点要被认定为「心智模型」而非「随口一说」，必须通过三重验证：

#### 验证1：跨域复现
同一思维框架出现在至少2个不同话题/领域。

**示例**：
```
候选："汇率影响不是线性的"

扫描所有视频：
- 视频A："人民币升值对出口的影响"
  → "升值不一定削弱出口，关键看竞争力来源"
  
- 视频B："中美贸易中的定价权"
  → "汇率变化对定价的影响取决于产业链位置"
  
- 视频C："产业升级的核心指标"
  → "不能简单用汇率解释出口变化"

结果：✅ 通过跨域验证（3个不同话题，同一框架）
```

#### 验证2：有生成力
用这个模型可以推断此人对新问题的可能立场。

**示例**：
```
模式："汇率影响不是线性的"

生成力测试：
问题：如果人民币贬值5%，对进口企业有什么影响？

基于模型的推断：
→ 不能简单说"成本上升5%"
→ 要看：产业链位置、定价权、替代品
→ 有定价权 → 转嫁成本
→ 无定价权 → 利润被挤压

结果：✅ 能对新问题生成合理推断
```

#### 验证3：有排他性
不是所有财经分析师都这样想，体现独特视角。

**示例**：
```
通用常识："汇率升值不利于出口"（所有人都知道）

刘德超独特视角：
- "汇率影响取决于竞争力来源，不是线性关系"
- 强调技术优势可以抵消汇率劣势
- 不是所有分析师都这样强调

结果：✅ 有区分度
```

**验证结果分类**：
- 三重通过 → **心智模型**
- 1-2重通过 → **决策启发式**
- 0重通过 → 丢弃

---

### 2. 表达DNA量化 ⭐⭐

量化"怎么说"，不只提取"说了什么"。

#### 提取维度

**句式指纹**：
- 平均句长
- 疑问句比例
- 类比密度（per 1000字）
- 确定性语气比例

**词汇特征**：
- 高频词（Top 15）
- 口头禅（3-5字高频短语）
- 确定性比例（"一定" vs "可能"）

**风格标签**：
```
正式 ←→ 口语
抽象 ←→ 具体
谨慎 ←→ 断言
学术 ←→ 通俗
```

#### 应用效果对比

❌ **未使用表达DNA**（通用AI风格）：
```
根据最新数据，当前经济形势可能呈现复杂态势。
从宏观指标来看，GDP增速或许会有所波动。
```

✅ **使用表达DNA**（刘德超风格）：
```
显然，从数据来看，当前经济形势很明确。
我们看到GDP增速5.2%，这个很稳定。
说白了，消费在恢复，但投资还弱。
```

**差异**：
- 使用高频词（"显然"、"从数据来看"）
- 短句为主
- 断言型语气（不用"可能"、"或许"）
- 口头禅（"说白了"）

```markdown
### 模型1：XXX
**一句话**：...
**证据**（跨域复现）：...
**应用**：...
**局限**：...

## 决策启发式（5-10条）

## 表达DNA

**句式特征**：...
**词汇特征**：...

## 价值观与反模式

**我追求的**：...
**我拒绝的**：...

## 诚实边界 ⭐

**这个Skill做不到的**：
- 无法预测全新话题的立场
- 公开表达 vs 真实想法可能有差距
- 信息截止到 [日期]
```

## 🚀 使用流程

### 前提：视频已转录

```bash
# 确保已完成 02_transcription 模块
# S3中应该有所有的 *_transcript.json 文件
```

### 执行认知合成

```bash
cd 03_cognitive_synthesis

# ⚠️ 必须配置环境变量（根据你的实际S3路径）
export S3_BUCKET=percyhan-douyin-video
export S3_PREFIX=post/刘德超/

# 执行合成
python3 cognitive_synthesis.py
```

**⚠️ 重要说明**：

1. **S3路径必须正确设置**：
   - 代码默认值是 `douyin_videos/20260417_150000/`（可能不是你的实际路径）
   - 必须通过环境变量 `S3_PREFIX` 设置为你的转录文件实际所在路径
   - 转录文件命名格式：`*_transcript.json`

2. **Screen/Tmux用户注意**：
   - 如果在 screen 或 tmux 中运行，每次重新连接会话后环境变量会丢失
   - 需要重新执行 `export` 命令

3. **验证S3路径**：
   ```bash
   # 运行前先确认转录文件存在
   aws s3 ls s3://${S3_BUCKET}/${S3_PREFIX} --recursive | grep transcript.json
   ```

### 工作流程

```
1. 从S3加载所有 *_transcript.json
   ↓
2. 提取候选思维模式（15-30个）
   使用Claude扫描所有视频，找反复出现的模式
   ↓
3. 三重验证（跨域、生成力、排他性）
   逐个验证每个候选，筛选出真正的心智模型
   ↓
4. 提取表达DNA
   量化句式、词汇、风格特征
   ↓
5. 提取价值观与反模式
   他追求什么、拒绝什么
   ↓
6. 构建诚实边界
   明确这个Skill做不到什么
   ↓
7. 生成SKILL.md
   按Claude Code Skill格式输出
```

## 💰 成本估算

### Claude调用成本

**步骤2：提取候选模式**
- 1次Claude Opus调用
- 输入：所有视频摘要（约10K tokens）
- 输出：15-30个候选（约5K tokens）
- 成本：~$0.50

**步骤3：三重验证**
- 假设30个候选
- 每个验证1次Claude Opus调用
- 成本：30 × $0.03 = $0.90

**总成本**：约$1.50（300个视频的认知合成）

**性价比**：
- 传统RAG：每次查询$0.001，但价值低（过时信息）
- 认知合成：一次性$1.50，生成可复用的Skill
- **价值提升10倍+**

## 🎯 质量标准（自检清单）

生成SKILL.md后，按以下标准自检：

### 心智模型
- [ ] 每个模型都有≥2个不同领域的证据？
- [ ] 模型数量3-7个？（太少=太浅，太多=没提炼）
- [ ] 每个模型都有明确的应用场景和局限？
- [ ] 模型之间有张力但不矛盾？

### 表达DNA
- [ ] 读起来有辨识度，不像通用AI？
- [ ] 没有过度模仿变成caricature？
- [ ] 抓住了核心特征而非表面模仿？

### 决策启发式
- [ ] 每条规则都有具体案例支撑？
- [ ] 可以被新情况触发（不只适用于原始案例）？

### 诚实边界
- [ ] 明确写了做不到什么？
- [ ] 标注了信息源和调研时间？
- [ ] 承认了信息不足的维度？

### 整体（最重要）
- [ ] **删掉名字后，还能认出这是谁的思维方式？** ⭐

## 🐛 故障排查

### 提取候选模式失败

```bash
# 检查S3访问权限
aws s3 ls s3://bucket/path/

# 检查转录文件数量
aws s3 ls s3://bucket/path/ --recursive | grep transcript.json | wc -l
```

### Bedrock权限错误

确保IAM角色包含：
```json
{
  "Effect": "Allow",
  "Action": [
    "bedrock:InvokeModel"
  ],
  "Resource": "arn:aws:bedrock:us-east-1::foundation-model/anthropic.claude-opus-*"
}
```

### 心智模型太少（<3个）

可能原因：
1. 视频数量不够（<20个）
2. 内容不够系统化（碎片化）
3. 验证标准太严格

解决方案：
- 增加视频数量
- 调整验证阈值
- 手动review候选模式
